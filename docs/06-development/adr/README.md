# Architecture Decision Records (ADR)

**Version:** 1.0
**Status:** Normative

На момент выпуска v1.0 документации GreenMarket эта папка пуста намеренно.

ADR — журнал решений, который начинает жить **после** начала разработки, а не часть исходной спецификации. Первый настоящий ADR появляется, когда возникает первое архитектурное отклонение или изменение уже согласованных нормативных документов настоящей документации.

## Формат записи

Один файл на решение: `NNNN-краткое-название.md` (например, `0001-perehod-na-edinuyu-migraciyu-schemy.md`), со структурой:

```markdown
# NNNN. Заголовок решения

**Дата:** YYYY-MM-DD
**Статус:** Предложено / Принято / Отклонено / Заменено ADR-XXXX

## Контекст
Какая проблема или ограничение потребовали решения.

## Решение
Что именно решено.

## Последствия
Что это меняет в документации, какие документы требуют правки.
```

Любое отклонение от нормативных документов ([DDL_Specification.md](../../03-database/DDL_Specification.md), [Coding_Standard.md](../../03-database/Coding_Standard.md), [Development_Guidelines.md](../Development_Guidelines.md) и др.) оформляется отдельным ADR — без этого изменение не считается согласованным.
