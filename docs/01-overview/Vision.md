# GreenMarket Vision

**Version:** 1.0
**Status:** Normative

## Назначение

Настоящий документ определяет цели, границы и основные принципы первого этапа проекта GreenMarket. Документ не описывает техническую реализацию системы и не содержит требований к отдельным компонентам. Его задача — зафиксировать единое понимание того, зачем создаётся проект и какие задачи решаются на первом этапе.

## Миссия проекта

GreenMarket создаёт единый цифровой каталог локального рынка, позволяющий покупателям быстро находить товары различных продавцов, а продавцам — публиковать свои предложения без сложных технических средств.

Проект не заменяет традиционный рынок. Проект делает рынок доступным в цифровой форме.

## Основная гипотеза

Локальные продавцы готовы регулярно публиковать свои каталоги, если процесс публикации достаточно прост.

Покупатели готовы использовать единый цифровой каталог, если информация в нём актуальна, достоверна и охватывает значительную часть рынка.

Проверка данной гипотезы является основной целью первого этапа.

## Цели Stage 1

Первый этап должен обеспечить:

- единый каталог товаров GreenMarket;
- единый каталог предложений всех продавцов;
- простую публикацию рабочих каталогов;
- поиск товаров покупателями;
- модерацию новых товарных позиций;
- получение первых реальных продавцов;
- получение первых реальных покупателей;
- получение первой регулярной выручки.

## Основные участники

В первом этапе участвуют четыре роли.

### Покупатель

Использует каталог для поиска товаров и выбора продавцов. Покупатель не оформляет заказ через платформу.

### Продавец

Ведёт собственный рабочий каталог в Microsoft Excel или Google Sheets. Публикует каталог через Publication Service.

### Администратор

Развивает единый каталог товаров. Выполняет модерацию новых товарных позиций. Активирует и деактивирует продавцов.

### Publication Service

Является единственным компонентом, изменяющим рабочий каталог продавца и состояние публикации.

## Что создаётся на первом этапе

Результатом первого этапа являются:

- единый каталог товаров;
- рабочие каталоги продавцов;
- Publication Service;
- единая база данных;
- Buyer Web;
- Seller Cabinet;
- Admin Cabinet.

## Критерии успешности

Первый этап считается успешным, если одновременно выполнены следующие условия:

- продавцы самостоятельно публикуют каталоги;
- покупатели регулярно используют каталог;
- каталог поддерживается в актуальном состоянии;
- система функционирует без ежедневного участия разработчиков;
- проект получает регулярную выручку.
