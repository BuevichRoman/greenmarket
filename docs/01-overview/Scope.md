# GreenMarket Stage 1 Scope

**Version:** 1.0
**Status:** Normative

## Назначение

Настоящий документ определяет функциональные границы первого этапа GreenMarket. Документ фиксирует, какие функции входят в Stage 1, какие сознательно исключены и какие являются предметом следующих этапов развития платформы.

## Цель Stage 1

Первый этап предназначен для проверки основной бизнес-гипотезы проекта. Необходимо подтвердить, что:

- продавцы готовы регулярно публиковать каталоги;
- покупатели готовы использовать единый каталог товаров;
- публикация каталога может выполняться без участия оператора;
- проект способен получать регулярную выручку.

## Функциональность Stage 1

### Единый каталог GreenMarket

Платформа предоставляет единый справочник товаров. Каталог используется для:

- классификации товаров;
- поиска;
- фильтрации;
- модерации;
- формирования общего каталога покупателей.

### Каталоги продавцов

Каждый продавец ведёт собственный каталог. Каталог включает:

- товар;
- цену;
- единицу измерения;
- остаток;
- описание;
- фотографии.

Рабочий каталог хранится у продавца в Microsoft Excel или Google Sheets (формат — [Catalog_Template.md](../02-domain/Catalog_Template.md)).

### Publication Service

Обеспечивает:

- импорт рабочего каталога;
- проверку данных;
- публикацию;
- обновление SellerProduct;
- обновление фотографий;
- создание записи CatalogPublication;
- обновление текущего состояния Seller.

Все изменения выполняются одной транзакцией.

### Модерация

Администратор выполняет:

- создание ProductGroup;
- создание Product;
- классификацию новых товаров;
- активацию продавцов;
- временную деактивацию продавцов.

До активации продавца его каталог недоступен покупателям.

### Buyer Web

Покупатель может:

- просматривать категории;
- искать товары;
- просматривать карточки товаров;
- видеть предложения различных продавцов.

### Seller Cabinet

Продавец может:

- скачать рабочий каталог;
- загрузить новую редакцию;
- просмотреть результаты публикации;
- просмотреть историю публикаций.

### Admin Cabinet

Администратор управляет:

- ProductGroup;
- Product;
- модерацией;
- продавцами.

## Что не входит в Stage 1

### Коммерческие процессы

Корзина, оформление заказа, оформление доставки, резервирование товаров, онлайн-оплата, возвраты, отмена заказов.

### Логистика

Маршрутизация, курьеры, управление доставкой, трекинг.

### Маркетинг

Акции, скидки, промокоды, бонусные программы, программа лояльности.

### Интеллектуальные функции

Рекомендации, подбор товаров, персонализация, прогнозирование, автоматическое составление корзины, AI-функции.

### Партии товара

Партии товара, отзывы по партиям, история партий, специальные характеристики отдельных партий продукции — см. [Future_Evolution.md](../02-domain/Future_Evolution.md).

## Ограничения Stage 1

Первый этап использует следующие принципы: один город; реальные продавцы; реальные товары; реальные цены; реальные остатки; ручное ведение рабочего каталога; один опубликованный каталог на продавца.

## Критерии завершения

Stage 1 считается завершённым при выполнении следующих условий:

- единая база данных введена в эксплуатацию;
- продавцы самостоятельно публикуют каталоги;
- покупатели используют каталог;
- каталог регулярно обновляется;
- система функционирует без ручного сопровождения публикаций;
- получена первая регулярная коммерческая выручка.

## Изменение Scope

Изменение настоящего документа допускается только: при изменении целей Stage 1; при утверждении нового этапа развития платформы; по Architecture Decision Record (ADR). Добавление новых функций без изменения настоящего документа не допускается.
