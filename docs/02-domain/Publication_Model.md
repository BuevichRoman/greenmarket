# GreenMarket Publication Model

**Version:** 1.0
**Status:** Normative

## Назначение

Настоящий документ определяет модель публикации рабочих каталогов продавцов в GreenMarket Stage 1. Документ описывает жизненный цикл публикации, ответственность Publication Service и правила обновления рабочего каталога продавца.

## Общая модель

Публикация представляет собой атомарную замену текущего рабочего каталога продавца новой редакцией. После успешной публикации новый каталог становится единственным актуальным каталогом продавца. Частичное обновление каталога не поддерживается.

## Источник данных

Единственным источником данных является рабочий каталог продавца. Поддерживаются форматы Microsoft Excel и Google Sheets (через экспорт в Excel) — точный формат определён в [Catalog_Template.md](Catalog_Template.md). Publication Service не предоставляет интерфейс ручного редактирования каталога.

## Publication Service

Publication Service является единственным компонентом системы, имеющим право изменять:

- SellerProduct;
- SellerProductPhoto;
- CatalogPublication;
- служебные поля Seller, связанные с публикацией каталога.

Прямое изменение этих данных другими компонентами не допускается.

## Основные этапы публикации

### 1. Загрузка каталога

Продавец загружает новую редакцию рабочего каталога.

### 2. Проверка файла

Выполняется проверка структуры, обязательных колонок, типов данных, формата. При наличии ошибок публикация завершается без изменения каталога.

### 3. Валидация данных

Проверяются обязательные поля, корректность цен, корректность остатков, существование фотографий, допустимые значения справочников. Ошибки возвращаются продавцу.

### 4. Сравнение

Publication Service сравнивает текущий каталог и новую редакцию. Определяются новые товары, изменённые товары, удалённые товары, неизменённые товары.

### 5. Обновление каталога

В рамках одной транзакции выполняется:

- создание новых SellerProduct;
- обновление существующих SellerProduct;
- снятие публикации с отсутствующих товаров;
- обновление фотографий;
- создание записи CatalogPublication;
- обновление текущего состояния Seller.

### 6. Завершение публикации

После успешного завершения транзакции новая редакция каталога становится текущей. Предыдущая редакция считается устаревшей.

## CatalogPublication

Каждая успешная публикация создаёт новую запись CatalogPublication. Запись содержит `publication_key`, `seller_id`, `version`, `hash`, дату публикации, техническую информацию.

CatalogPublication представляет собой журнал публикаций. Записи журнала не изменяются после создания.

## Текущее состояние каталога

Текущее состояние каталога хранится в сущности Seller. Publication Service обновляет `current_catalog_version`, `current_publication_key`, `current_catalog_hash`. Эти поля используются для быстрого определения актуальной версии каталога.

## Атомарность

Все изменения выполняются одной транзакцией базы данных. Публикация считается успешной только после успешной фиксации всей транзакции. При любой ошибке выполняется полный откат изменений.

## Идемпотентность

Каждая публикация идентифицируется уникальным `publication_key`. Повторная обработка публикации с тем же ключом не допускается. Publication Service обязан корректно обрабатывать повторную отправку одного и того же запроса. Правила генерации и ротации `publication_key` на стороне клиента — см. [Catalog_Template.md](Catalog_Template.md).

## История публикаций

История публикаций сохраняется полностью. Удаление записей CatalogPublication не допускается. Изменение существующих записей не допускается. История используется для аудита, диагностики, анализа публикаций.

## Ограничения модели

Publication Service не выполняет:

- модерацию товаров;
- изменение Product;
- изменение ProductGroup;
- оформление заказов;
- резервирование товаров;
- изменение данных покупателей.

Ответственность сервиса ограничивается публикацией рабочего каталога продавца.

## Основные принципы

Модель публикации строится на следующих принципах.

- Рабочий каталог продавца является единственным источником данных.
- Publication Service является единственным владельцем процесса публикации.
- Каждая публикация полностью заменяет текущую редакцию каталога.
- История публикаций отделена от текущего состояния.
- Все изменения выполняются одной транзакцией.
- Публикация должна быть идемпотентной.
- После успешной публикации каталог становится доступным покупателям только при активном статусе продавца.
