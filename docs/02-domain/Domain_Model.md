# GreenMarket Domain Model

**Version:** 1.0
**Status:** Normative

## Назначение

Настоящий документ определяет предметную модель подсистемы каталога GreenMarket Stage 1. Документ описывает основные бизнес-сущности, их назначение, ответственность и взаимосвязи.

Документ не определяет физическую модель базы данных и не содержит требований к реализации.

## Назначение предметной модели

Предметная модель разделяет платформу на независимые бизнес-сущности. Каждая сущность имеет единственную ответственность и собственный жизненный цикл.

## Основные сущности

Подсистема каталога Stage 1 использует следующие сущности.

Платформенные (уже существуют на платформе iBronevik): Seller, User, Photo.

Новые: ProductGroup, Product, SellerProduct, SellerProductPhoto, CatalogPublication.

## ProductGroup

**Назначение:** дерево товарных групп GreenMarket. Группа используется для навигации, поиска, фильтрации, классификации товаров.

**Ответственность:** ProductGroup отвечает только за структуру каталога. Группа не содержит коммерческих данных.

**Жизненный цикл:** создаётся администратором. Редко изменяется. Физически не удаляется при наличии связанных товаров.

## Product

**Назначение:** единая товарная позиция платформы. Например: Апельсин, Банан, Огурец, Помидор.

**Ответственность:** Product определяет только товар. Product не содержит: цены, остатков, фотографий, описания продавца, коммерческих характеристик.

**Жизненный цикл:** создаётся один раз. Используется всеми продавцами. При необходимости выводится из использования через `is_active`.

## SellerProduct

**Назначение:** конкретное предложение конкретного продавца. Это основная рабочая сущность подсистемы.

**Ответственность:** содержит продавца, цену, остаток, описание, единицу измерения, статус публикации, статус модерации.

**Особенности:** связь с Product может отсутствовать до завершения модерации. В этом случае Product определяется администратором позднее.

**Жизненный цикл:** создаётся первой публикацией. Далее изменяется только Publication Service. Физически не удаляется. При исключении из каталога переводится в состояние `is_published = false`.

## SellerProductPhoto

**Назначение:** связывает товар продавца с фотографиями.

**Ответственность:** определяет состав фотографий, порядок отображения. Других бизнес-данных не содержит.

## CatalogPublication

**Назначение:** журнал публикаций рабочих каталогов продавцов.

**Ответственность:** хранит только сведения о факте успешной публикации. Не хранит содержимое опубликованного каталога. Не определяет текущее состояние каталога.

**Жизненный цикл:** каждая успешная публикация создаёт новую запись. Изменение существующих записей не допускается. Удаление записей не допускается.

## Seller

**Назначение:** зарегистрированный продавец платформы. Подсистема каталога использует существующую платформенную сущность.

**Дополнительные данные:** для поддержки Publication Service используются `current_catalog_version`, `current_publication_key`, `current_catalog_hash`. Эти поля описывают текущее состояние рабочего каталога.

## User

Используется для регистрации действий пользователей платформы. Подсистема каталога не изменяет модель User.

## Photo

Представляет фотографии платформы. Подсистема каталога использует существующую сущность без изменения модели.

## Основные связи

```text
ProductGroup
│
▼
Product
│
▼
SellerProduct
│
▼
SellerProductPhoto
```

Отдельно существует журнал публикаций.

```text
Seller
│
▼
CatalogPublication
```

## Принципы модели

Предметная модель строится на следующих принципах.

- Единый каталог отделён от каталогов продавцов.
- Product не содержит коммерческой информации.
- SellerProduct содержит только коммерческие данные конкретного продавца.
- История публикаций отделена от текущего состояния.
- Publication Service является владельцем служебных данных.
- Все изменения рабочего каталога выполняются одной транзакцией.
