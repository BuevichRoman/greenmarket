# GreenMarket Catalog Model

**Version:** 1.0
**Status:** Normative

## Назначение

Настоящий документ определяет логическую модель каталога GreenMarket Stage 1. Документ описывает структуру единого каталога платформы, каталогов продавцов и правила их взаимодействия.

## Общая модель

Подсистема каталога состоит из двух независимых частей: единого каталога GreenMarket и рабочих каталогов продавцов.

Единый каталог определяет классификацию товаров. Рабочие каталоги содержат коммерческие предложения конкретных продавцов.

## Единый каталог GreenMarket

Единый каталог представляет собой централизованный справочник товаров платформы. Каталог используется для:

- классификации товаров;
- поиска;
- фильтрации;
- построения навигации;
- модерации;
- аналитики.

## Структура каталога

Каталог имеет древовидную структуру. Пример:

```text
Фрукты
  Цитрусовые
    Апельсин
    Лимон
    Мандарин
  Яблоки
    Голден
    Гренни Смит
Овощи
  Томаты
  Огурцы
  Картофель
Молочные продукты
  Молоко
  Сыр
  Йогурты
```

Каждый Product принадлежит одной ProductGroup.

## ProductGroup

ProductGroup определяет структуру дерева. Группа не содержит коммерческой информации. Группы создаются и сопровождаются администраторами платформы.

## Product

Product представляет единую товарную позицию платформы. Каждая позиция создаётся один раз. После создания используется всеми продавцами.

Например: апельсин существует в системе в единственном экземпляре независимо от количества продавцов.

## Каталог продавца

Каждый продавец имеет собственный независимый каталог. Каталог состоит из SellerProduct. SellerProduct связывает продавца с Product и содержит коммерческие характеристики предложения. Коммерческое наименование продавца может отличаться от единой позиции GreenMarket (например, «Orange Premium» и «Апельсин Марокко» — оба ссылаются на Product «Апельсин»).

## Коммерческие данные

Коммерческие характеристики принадлежат исключительно SellerProduct. К ним относятся:

- цена;
- остаток;
- единица измерения;
- описание;
- фотографии;
- статус публикации.

Эти данные никогда не хранятся в Product.

## Новые товары

Если продавец не находит подходящий Product: выбирается ProductGroup, Product остаётся пустым, каталог публикуется. Publication Service создаёт SellerProduct без связи с Product (`product_id = NULL`). Такой товар поступает в очередь модерации со статусом `moderation_status = WAIT_PRODUCT`.

## Модерация

Администратор выполняет одно из действий: связывает SellerProduct с существующим Product; создаёт новый Product; при необходимости создаёт новую ProductGroup.

После завершения модерации новый Product становится частью единого каталога — все продавцы получают его при следующем обновлении рабочего каталога.

## Развитие каталога

Единый каталог развивается постепенно. Начальная версия содержит только минимально необходимый набор товарных групп и товаров. Расширение каталога выполняется администраторами платформы. Publication Service не изменяет структуру единого каталога.

## Удаление

ProductGroup физически не удаляется при наличии связанных Product. Product физически не удаляется при наличии связанных SellerProduct. SellerProduct физически не удаляется.

Удаление предложения выполняется переводом записи в состояние `is_published = false`. Это обеспечивает сохранение SellerProductId, результатов модерации, фотографий, ссылочной целостности.

## Принципы модели

Каталог строится на следующих принципах.

- Один Product используется всеми продавцами.
- Product не содержит коммерческой информации.
- SellerProduct содержит только данные конкретного продавца.
- Каталог развивается как данные платформы, а не как изменение структуры БД.
- Publication Service не изменяет единый каталог.
- Единый каталог развивается только через административную модерацию.
