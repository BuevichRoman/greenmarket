# GreenMarket Publication Service

**Version:** 1.0
**Status:** Normative

## Назначение

Publication Service обеспечивает публикацию рабочих каталогов продавцов. Сервис является единственным компонентом системы, имеющим право изменять опубликованный каталог продавца и связанную с ним служебную информацию. Все остальные компоненты работают только с результатом публикации.

## Ответственность

Publication Service отвечает за:

- импорт рабочего каталога;
- проверку структуры файла;
- валидацию данных;
- сравнение редакций каталога;
- создание новых SellerProduct;
- обновление существующих SellerProduct;
- снятие с публикации отсутствующих товаров;
- обновление фотографий;
- создание записи CatalogPublication;
- обновление текущего состояния Seller.

Publication Service не отвечает за:

- модерацию товаров;
- изменение Product;
- изменение ProductGroup;
- поиск товаров;
- отображение каталога;
- оформление заказов.

## Владение данными

Publication Service является единственным владельцем следующих данных.

**Таблицы:** SellerProduct, SellerProductPhoto, CatalogPublication.

**Поля Seller:** `current_catalog_version`, `current_publication_key`, `current_catalog_hash`.

Изменение данных другими сервисами запрещается.

## Источник данных

Источник данных только один — статическая таблица Google Sheets, созданная продавцом как копия шаблона GreenMarket (CR-001); точный формат — [Catalog_Template.md](../02-domain/Catalog_Template.md). Ручное изменение опубликованного каталога через интерфейс не допускается.

### Google Sheets Parser

`GoogleSheetsParser` читает таблицу через Service Account (`spreadsheets.values.batchGet`, `valueRenderOption=UNFORMATTED_VALUE` — обязательное условие эквивалентности `ExcelParser`). Таймаут запроса к Google API — 10 секунд, без retry в Stage 1 (публикация — синхронное пользовательское действие; лучше быстрый отказ, чем зависший HTTP-запрос). Контракт `GoogleSheetsParser` идентичен `ExcelParser` — возвращает тот же `RawWorkbook`, не вычисляет `PublicationKey`/`CatalogHash`.

### PublicationKey и CatalogHash — генерация сервером

`PublicationKey` (`uuid.uuid4()`) и `CatalogHash` (SHA-256 от содержимого `RawWorkbook`, посчитанный до `Validator`) больше не читаются из документа — оба генерируются/вычисляются сервером на каждый вызов публикации (CR-001).

## Общий процесс публикации

```text
Загрузка файла → Проверка структуры → Валидация → Сравнение →
Обновление каталога → Создание CatalogPublication → Обновление Seller → Commit
```

Все этапы выполняются в рамках одной транзакции.

## Проверка структуры

Перед обработкой файла проверяются: обязательные листы, обязательные колонки, поддерживаемая версия шаблона. При ошибке публикация прекращается.

## Валидация данных

Проверяются:

- обязательные значения;
- типы данных;
- цена;
- остаток;
- единицы измерения;
- ссылки на фотографии.

Ошибки собираются в единый отчёт и возвращаются продавцу. До исправления ошибок публикация не выполняется.

## Сравнение редакций

Publication Service сравнивает опубликованный каталог и новую редакцию. Для каждой позиции определяется одно из состояний: `NEW`, `UPDATED`, `UNCHANGED`, `REMOVED`. Сопоставление строк выполняется по скрытому техническому идентификатору `SellerProductId` (см. [Catalog_Template.md](../02-domain/Catalog_Template.md)) — не по коммерческому наименованию.

## Обновление SellerProduct

В рамках одной транзакции выполняются:

- создание новых записей;
- изменение существующих;
- снятие публикации с отсутствующих товаров;
- обновление коммерческих данных;
- обновление фотографий.

Физическое удаление записей не используется.

## Создание CatalogPublication

После успешного обновления каталога создаётся запись в журнале публикаций:

- `publication_key`;
- `seller_id`;
- `version`;
- `catalog_hash`;
- дата публикации.

CatalogPublication является append-only журналом — после создания запись не изменяется.

## Обновление Seller

После успешной публикации обновляются поля `current_catalog_version`, `current_publication_key`, `current_catalog_hash` — текущее состояние каталога продавца.

## Атомарность

Публикация — атомарная операция. Все изменения выполняются одной транзакцией. При любой ошибке выполняется полный rollback. Частично опубликованный каталог существовать не может.

## Выбор синхронной обработки

Publication выполняется синхронно в рамках одного HTTP-запроса и одной транзакции базы данных. Публикация не имеет собственного долгоживущего процесса или промежуточных статусов — ответ на запрос содержит либо успешный результат, либо полный список ошибок.

В Stage 1 публикация выполняется синхронно, поскольку ожидаемый объём каталогов позволяет завершить обработку в пределах одного HTTP-запроса. Переход к асинхронной обработке рассматривается только при появлении подтверждённой необходимости и не требует изменения предметной модели.

## Идемпотентность

Каждая публикация имеет уникальный `publication_key`. Повторная обработка публикации с тем же ключом запрещается. Publication Service обязан корректно обрабатывать повторную отправку одинакового запроса.

## История публикаций

История публикаций не изменяется. Запрещается `UPDATE`/`DELETE` CatalogPublication. История используется для аудита, диагностики, анализа публикаций.

## Доступность каталога

После успешной публикации каталог становится доступен покупателям только при выполнении двух условий: Seller активирован; Seller не находится в состоянии временной деактивации. Publication Service не принимает решение о доступности каталога — он только публикует данные. Решение принимает бизнес-состояние Seller.

## Ошибки публикации

Ошибки подразделяются на: ошибки структуры, ошибки валидации, технические ошибки. При любой ошибке состояние опубликованного каталога не изменяется.

## Производительность

Publication Service должен обеспечивать обработку каталога без деградации пользовательского опыта. Оптимизация выполняется после анализа реальных каталогов и измерения производительности. Преждевременная оптимизация не допускается.

## Основные принципы

Publication Service строится на следующих принципах: один владелец данных; один источник данных; одна транзакция; полная атомарность; идемпотентность; неизменяемая история публикаций; отделение истории от текущего состояния.
