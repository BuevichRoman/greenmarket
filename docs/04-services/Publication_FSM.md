# GreenMarket Publication FSM

**Version:** 1.0
**Status:** Normative

## Назначение

Настоящий документ определяет конечный автомат (FSM), управляющий жизненным циклом **записи публикации** на сервере — то есть обработкой одной загрузки рабочего каталога от момента приёма файла до фиксации результата.

FSM описывает только процесс публикации каталога. Жизненный цикл Product, SellerProduct и Seller определяется отдельными документами. Состояние клиентского документа (рабочего каталога Excel) и механизм `PublicationKey` описаны в [Catalog_Template.md](../02-domain/Catalog_Template.md).

## Назначение FSM

Publication FSM обеспечивает: однозначное состояние публикации; допустимые переходы; атомарность бизнес-процесса; единообразную обработку ошибок. Все публикации проходят одинаковую последовательность состояний.

## Состояния

- **NEW** — публикация зарегистрирована, файл ещё не обработан.
- **VALIDATING** — выполняется проверка структуры, версии шаблона, обязательных колонок.
- **VALIDATION_FAILED** — обнаружены ошибки структуры или данных; каталог продавца не изменяется; публикация считается завершённой с ошибкой.
- **PROCESSING** — бизнес-обработка: сравнение каталогов, создание/обновление SellerProduct, обновление фотографий, создание CatalogPublication, обновление Seller. Все операции — одной транзакцией.
- **FAILED** — во время обработки произошла техническая ошибка; выполнен полный rollback; каталог продавца остаётся без изменений.
- **COMPLETED** — публикация успешно завершена; новая версия каталога становится текущей.

## Диаграмма состояний

```text
NEW
 │
 ▼
VALIDATING
 │       │
 ▼       ▼
PROCESSING   VALIDATION_FAILED
 │       │
 ▼       ▼
COMPLETED   FAILED
```

## Допустимые переходы

| Текущее состояние | Следующее состояние |
|---|---|
| NEW | VALIDATING |
| VALIDATING | PROCESSING |
| VALIDATING | VALIDATION_FAILED |
| PROCESSING | COMPLETED |
| PROCESSING | FAILED |

Другие переходы запрещены.

## Терминальные состояния

Публикация считается завершённой в состояниях `VALIDATION_FAILED`, `FAILED`, `COMPLETED`. После достижения терминального состояния переходы невозможны. Для повторной публикации создаётся новая запись.

## Rollback

Rollback выполняется только из состояния `PROCESSING`. Причины:

- ошибка SQL;
- нарушение ограничений;
- системная ошибка;
- ошибка хранения файлов;
- внутренняя ошибка Publication Service.

После rollback состояние каталога полностью соответствует состоянию до начала публикации.

## Ошибки

Ошибки делятся на две категории.

**Ошибки пользователя** (отсутствует обязательная колонка, неверный формат цены, некорректная единица измерения) → `VALIDATION_FAILED`.

**Технические ошибки** (ошибка БД, ошибка файловой системы, ошибка сети, внутренняя ошибка сервиса) → `FAILED`.

## Повторная публикация

После любого терминального состояния продавец может загрузить новый файл. Повторная публикация создаёт новую запись Publication. FSM предыдущей публикации не изменяется.

## Инварианты

Publication FSM гарантирует: одна публикация имеет только одно состояние; запрещены циклические переходы; завершённая публикация не изменяется; опубликованный каталог не бывает частично обновлён; rollback полностью восстанавливает предыдущее состояние.
