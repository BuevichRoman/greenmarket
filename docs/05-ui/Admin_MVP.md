# GreenMarket Admin Cabinet MVP

**Version:** 1.0
**Status:** Normative

## Назначение

Настоящий документ определяет минимальный административный интерфейс GreenMarket Stage 1. Admin Cabinet предназначен для сопровождения единого каталога платформы и управления продавцами. Административный интерфейс не используется для сопровождения коммерческих операций, заказов или доставки.

## Цель MVP

Admin Cabinet должен обеспечить возможность сопровождения единого каталога без участия разработчиков. Администратор должен иметь возможность: управлять деревом категорий; управлять единым каталогом товаров; выполнять модерацию новых товаров; управлять состоянием продавцов.

## Общие принципы

Простота; минимальное количество экранов; все изменения выполняются через REST API; каждая операция имеет единственную ответственность.

## Авторизация

Доступ к Admin Cabinet имеют только пользователи с административными правами. Все действия администратора должны журналироваться платформой.

## Основные функции

Управление ProductGroup; управление Product; модерация новых SellerProduct; активация продавцов; временная деактивация продавцов.

## Экран 1. Product Groups

Администратор может создавать группы, изменять названия, изменять положение в дереве, отключать группы. Удаление группы при наличии связанных Product не допускается.

## Экран 2. Products

Администратор может создавать Product, изменять свойства Product, переносить Product между ProductGroup, деактивировать Product. Удаление Product при наличии связанных SellerProduct не допускается.

## Экран 3. Moderation Queue

Отображаются SellerProduct, для которых отсутствует связь с Product (`moderation_status = WAIT_PRODUCT`). Для каждой позиции администратор может: связать с существующим Product; создать новый Product; при необходимости создать новую ProductGroup. После завершения модерации товар становится частью единого каталога.

## Экран 4. Sellers

Отображается список продавцов. Для каждого продавца доступны операции: активация, временная деактивация, просмотр основной информации.

## Активация продавца

После регистрации продавец имеет состояние «Pending Activation». До активации: каталог может публиковаться, публикации сохраняются, каталог не отображается покупателям. После активации последняя опубликованная версия каталога становится доступной покупателям автоматически.

## Временная деактивация

При временной деактивации:

- публикации сохраняются;
- SellerProduct сохраняются;
- история публикаций сохраняется;
- каталог скрывается от покупателей.

Повторная активация восстанавливает доступность последней опубликованной версии без повторной публикации.

## Ограничения

Admin Cabinet не позволяет: изменять SellerProduct вручную; изменять CatalogPublication; изменять служебные поля Publication Service; публиковать каталог вместо продавца. Все изменения каталога продавца выполняются исключительно Publication Service.

## Источник данных

Admin Cabinet работает только через REST API. Прямой доступ к базе данных запрещён.

## Definition of Done

Admin MVP считается завершённым, если администратор способен: сопровождать дерево ProductGroup; сопровождать Product; обработать очередь модерации; активировать нового продавца; временно деактивировать продавца; вернуть продавца в активное состояние без повторной публикации каталога.

## Границы MVP

В Stage 1 отсутствуют: управление заказами; управление доставкой; управление оплатой; управление пользователями платформы; аналитика продаж; финансовая отчётность. Данные функции относятся к следующим этапам развития GreenMarket.
