# GreenMarket Buyer Web MVP

**Version:** 1.0
**Status:** Normative

## Назначение

Настоящий документ определяет минимальный пользовательский интерфейс покупателя GreenMarket Stage 1. Buyer Web предназначен исключительно для поиска и просмотра товаров локального рынка. Покупатель не оформляет заказ через систему.

## Цель MVP

Buyer Web должен подтвердить основную гипотезу проекта. Покупатель способен: найти нужный товар; увидеть предложения разных продавцов; сравнить предложения; выбрать продавца. На этом задачи первого этапа заканчиваются.

## Общие принципы

Простота; высокая скорость работы; минимальное количество экранов; отсутствие лишних действий; Mobile First.

## Авторизация

В Stage 1 просмотр каталога не требует регистрации. Все товары доступны анонимному пользователю.

## Экраны

### Экран 1. Главная

Начальная точка работы покупателя. Основные элементы:

- строка поиска;
- дерево категорий;
- популярные категории;
- последние опубликованные товары (опционально).

### Экран 2. Каталог товаров

Отображение списка товаров. Покупатель может искать, выбирать категорию, просматривать результаты. Каждый товар отображает фотографию, название, минимальную цену, количество продавцов.

### Экран 3. Карточка товара

Отображение предложений продавцов. Для каждого предложения:

- продавец;
- цена;
- единица измерения;
- остаток;
- фотографии;
- описание товара продавца.

## Поиск

Поиск выполняется по Product. Результаты группируются по товару. Если несколько продавцов предлагают один Product, покупатель сначала видит товар, а затем список предложений.

## Категории

Навигация осуществляется по ProductGroup. Структура категорий определяется единым каталогом GreenMarket.

## Предложения продавцов

Отображаются только предложения, удовлетворяющие всем условиям: Seller активирован; Seller не деактивирован; SellerProduct опубликован; Product активен. Неопубликованные предложения не отображаются.

## Фотографии

Используются фотографии SellerProduct. Если фотографии отсутствуют, отображается изображение-заглушка платформы.

## Остатки

Если продавец ведёт остатки, они отображаются покупателю. Если остатки не публикуются, интерфейс не отображает соответствующую информацию.

## Цены

Покупателю всегда отображается актуальная опубликованная цена продавца. История изменения цен в Stage 1 отсутствует.

## Сортировка

Минимально поддерживаются: по названию, по цене, по популярности (при наличии данных). Дополнительные способы сортировки относятся к следующим этапам.

## Не входит в Buyer MVP

Корзина, оформление заказа, онлайн-оплата, доставка, избранное, история покупок, отзывы, рейтинг продавцов, рекомендации, персонализация.

## Источник данных

Buyer Web работает исключительно через REST API. Прямой доступ к базе данных запрещён. Все данные являются результатом работы Publication Service.

## Требования к производительности

Быстрое открытие каталога, быстрый поиск, плавная навигация между экранами. Оптимизация выполняется на основании результатов эксплуатации.

## Definition of Done

Buyer MVP считается завершённым, если пользователь способен без регистрации: открыть каталог; найти товар; открыть карточку товара; сравнить предложения нескольких продавцов; выбрать подходящего продавца.
